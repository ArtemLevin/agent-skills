# Contributing

## Добавление навыка

Каждый навык располагается в `skills/<name>/SKILL.md` и обязан содержать YAML frontmatter:

```yaml
---
name: example-skill
description: >
  Use when ...
---
```

Требования:

- имя в `kebab-case` и совпадает с именем директории;
- описание содержит конкретные условия активации;
- навык имеет разделы `Purpose`, `Inputs`, `Workflow`, `Decision rules`, `Output`, `Stop conditions`;
- инструкции описывают решения и ограничения, а не общие пожелания;
- основной файл остаётся компактным; большие справочники выносятся в `references/`;
- навык не дублирует глобальные правила из `AGENT.md` без необходимости.

## Изменение политики

Изменения в `policies/` должны быть совместимы со всеми навыками. При изменении структуры выходных данных обновите соответствующую JSON Schema и пример в `examples/`.

## Проверка

```bash
python scripts/validate_skills.py
python scripts/validate_release.py
python scripts/scan_secrets.py
python -m unittest discover -s tests
```

Перед PR проверьте:

- отсутствуют конфликтующие инструкции;
- условия активации навыка различимы;
- новый навык не запускается для каждой задачи без причины;
- критерии остановки сформулированы явно;
- примеры не содержат секретов и привязки к частной инфраструктуре.
