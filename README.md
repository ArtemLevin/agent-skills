# Agent Skills Engineering Kit

Комплексная обвязка coding-агента узкими навыками для повышения качества генерации кода без неконтролируемого расхода токенов.

Проект реализует модель:

```text
один управляющий агент
+ динамически подключаемые навыки
+ минимальный контекст
+ риск-ориентированная верификация
+ независимое критическое ревью
+ строгий критерий остановки
```

Внутренние инструкции навыков написаны на английском: это облегчает перенос между агентными платформами и обычно сокращает токенизацию. Документация для внедрения приведена на русском языке.

## Что входит

- `AGENT.md` — глобальный контракт управляющего агента;
- `skills/` — 15 узких навыков планирования, реализации, тестирования и ревью;
- `policies/` — общие quality gates, модель серьёзности и токен-бюджет;
- `schemas/` — JSON Schema для машинно-проверяемых результатов этапов;
- `templates/` — шаблоны плана, ADR, ревью и финального отчёта;
- `scripts/` — валидатор навыков и экономный построитель карты репозитория;
- `tests/` и GitHub Actions — автоматическая проверка структуры.

## Базовый маршрут

```text
request
  -> task-triage
  -> repository-context
  -> requirements-contract
  -> change-planner (when needed)
  -> architecture-guard / engineering-balance (when needed)
  -> implementation
  -> verification-router
  -> risk-based-testing
  -> specialist reviews (when triggered)
  -> adversarial-review
  -> targeted fix
  -> intent-documentation
  -> delivery-summary
```

Не каждый навык запускается для каждой задачи. `task-triage` выбирает режим и активирует только релевантные компоненты.

## Режимы

| Режим | Когда использовать | Типичный маршрут |
|---|---|---|
| `fast` | Очевидная локальная правка с низким риском | context → implement → targeted check → summary |
| `standard` | Обычный bugfix или небольшая функция | contract → plan → implement → tests → review |
| `deep` | БД, auth, concurrency, публичный API, production | contract → architecture → implement → specialist checks → adversarial review |

## Быстрый старт

1. Скопируйте `AGENT.md`, `policies/`, `skills/`, `schemas/` и `templates/` в каталог конфигурации вашего агента.
2. Дайте агенту доступ только к тем инструментам, которые разрешены в `policies/tool-permissions.md`.
3. На входе каждой задачи запускайте `task-triage`.
4. Перед завершением требуйте прохождения `completion_gate` из `policies/quality-gates.md`.
5. Проверяйте репозиторий:

```bash
python scripts/validate_skills.py
python -m unittest discover -s tests
```

Для построения компактной карты целевого проекта:

```bash
python scripts/repo_map.py /path/to/project --max-depth 4
```

## Принципы

1. **Минимальный полезный diff.** Изменение должно полностью решать задачу и не затрагивать несвязанный код.
2. **Доказательства вместо уверенности.** Агент не объявляет проверку успешной, если команда не выполнялась.
3. **Тесты по риску, не по проценту.** Новый тест обязан защищать поведение или инвариант.
4. **Комментарии объясняют причины.** Комментарий отвечает «почему это необходимо», а не пересказывает синтаксис.
5. **Контекст загружается постепенно.** Сначала дерево и символы, затем только релевантные файлы.
6. **Остановка обязательна.** P3-полировка и гипотетические улучшения не продлевают задачу.

## Совместимость

Структура `skills/<name>/SKILL.md` использует YAML frontmatter с полями `name` и `description` и следует принципу progressive disclosure: полное содержание навыка читается только после его активации.
